Introduction

การเรียงลำดับข้อมูล (Sorting)

การเรียงลำดับข้อมูล (Sorting) คือ การนำข้อมูลมาดำเนินการประมวลผลให้ได้ข้อมูลใหม่ที่มีการจัดเรียงใหม่ ซึ่งผลการจัดเรียงอาจเรียงข้อมูลจากน้อยไปมาก (Ascending) หรือมากไปน้อย (Descending)

ข้อดีของการเรียงลำดับข้อมูล (Advantages of Sorting)

- ลดเวลาในการค้นหาข้อมูล ช่วยให้ค้นหาข้อมูลได้ง่ายมากขึ้น
- ช่วยให้การจัดเก็บข้อมูลเป็นระเบียบมากขึ้น
- ช่วยให้วิเคราะห์ข้อมูลได้เร็ว และง่ายขึ้น

ข้อเสียของการเรียงลำดับข้อมูล (Disadvantages of Sorting)

- ใช้เวลาในการเรียงข้อมูลนานเมื่อมีข้อมูลมาก
- ใช้พื้นที่ในการจัดเก็บข้อมูลเกินจำเป็นเมื่อมีข้อมูลมาก
- แก้ไขข้อมูลที่มีอยู่ในระบบยากขึ้น
- เกิดข้อผิดพลาดในการเรียงลำดับข้อมูลได้ง่าย

Example

Explain Algorithm

การเรียงลำดับแบบผสาน (Merge Sort)

การเรียงลำดับแบบผสาน (Merge Sort) จะใช้หลักการการแบ่งแยกและเอาชนะ (Divide and conquer) ซึ่งลักษณะการทำงานคือจะนำข้อมูลมาแบ่งออกเป็นส่วน ๆ 2 ส่วนเรื่อย ๆ จนกว่าจะแบ่งไม่ได้ แล้วเรียงข้อมูลในส่วนย่อย จากนั้นก็นำข้อมูลส่วนย่อยที่เรียงไว้มารวมกัน (merge) และเรียงลำดับไปในเวลาเดียวกัน

Time Complexity

Best Case Complexity: O(n*log n)
Worst Case Complexity: O(n*log n)
Average Case Complexity: O(n*log n)

ข้อดีของการเรียงลำดับข้อมูลแบบผสาน (Advantages of Merge Sort)

- เหมาะกับการเรียงข้อมูลขนาดใหญ่
- เป็นอัลกอริทึมที่มีขั้นตอนการทำงานที่เรียบง่าย และเข้าใจง่าย
- เป็นอัลกอริทึมที่มีความเสถียร (stable) ลำดับของข้อมูลที่ไม่ซ้ำจะไม่เปลี่ยนแปลงหลังจากการเรียงลำดับ
- มีเวลาในการทำงานเป็น O(n*log n) ในทุกกรณี

ข้อเสียของการเรียงลำดับข้อมูลแบบผสาน (Disadvantages of Merge Sort)

- ประสิทธิภาพอาจลดลงหากข้อมูลมีลำดับที่ค่อนข้างดีอยู่แล้ว
- การทำงานอาจช้ากว่าอัลกอริทึมการเรียงลำดับแบบอื่น ๆ ในกรณีที่มีข้อมูลขนาดเล็ก
- การทำงานอาจต้องใช้หน่วยความจำมากขึ้นในการจัดเก็บข้อมูลชั่วคราวสำหรับการเรียงลำดับข้อมูล

สรุป

เป็นอัลกอริทึมการเรียงลำดับที่มีประสิทธิภาพสูง ทำงานได้เสถียร และไม่จำเป็นต้องใช้หน่วยความจำเพิ่มเติมในการเรียงลำดับข้อมูล แต่อาจทำงานช้ากว่าอัลกอริทึมการเรียงลำดับแบบอื่น ๆ ในกรณีของข้อมูลขนาดเล็ก และมี Time Complexity เป็น O(n*log n) ในทุกกรณี

Example

ตัวอย่าง

การเรียงลำดับข้อมูลจากน้อยไปหามาก ดังนี้ (7, 4, 3, 1) ด้วยขั้นตอนการเรียงแบบ Merge Sort

Step by step

Step 1: นำข้อมูล (7, 4, 3, 1) มาแยกเป็น 2 ส่วน โดยใช้สูตรที่พิจารณาจากดัชนี (3+0)/2 = 1 (ตัดเศษทิ้ง) ดังนั้น ข้อมูลที่ 0 - 1 จะเป็นก้อนแรก และข้อมูลที่ 2 - 3 จะเป็นก้อนที่สอง

Step 2: นำข้อมูลก้อนแรกมาแยกเป็นสองส่วน จะได้ 7 และ 4 และนำข้อมูลก้อนที่สอง มาแยกเป็นสองส่วน จะได้เป็น 3 และ 1

Step 3: นำข้อมูลที่ได้มารวมกันทีละก้อน โดยเปรียบเทียบจากก้อนแรก 7 > 4 จึงสลับที่ และได้ผลลัพธ์เป็น 4, 7 และก้อนที่สอง 3 > 1 จึงสลับที่กัน จะได้ผลลัพธ์เป็น 1, 3

Step 4: นำข้อมูลก้อนแรก 7, 4 และก้อนที่สอง 3, 1 มารวมกัน โดยเปรียบเทียบจากซ้ายไปขวาทีละตัว แล้วแทรกเข้าไปทีละตัวให้ถูกตำแหน่ง เพื่อให้ได้แถวลำดับใหม่ จะได้ผลลัพธ์เป็น 1, 3, 4, 7

Application - Benefit

- Merge sort เหมาะที่จะนำไปใช้กับการเรียงลำดับที่มีชุดข้อมูลขนาดใหญ่
- นำมาประยุกต์ใช้กับ external sorting
- นำมาประยุกต์ใช้กับ Linked lists
- นำมาประยุกต์ใช้กับการนับแบบผกผัน (counting inversions)

Counting Inversions Using Merge Sort
Time Complexity

Counting Inversions : O(n^2)
Count Inversions Using MergeSort : O(n*log n)
